# 4.7.1 Implementasi Antarmuka Pengguna

Implementasi antarmuka pengguna dilakukan untuk menampilkan fitur-fitur sistem e-commerce yang dapat digunakan oleh pengguna, baik customer, seller, maupun admin. Antarmuka sistem dirancang berdasarkan hasil perancangan use case dan activity diagram yang telah dibuat sebelumnya.

Sistem Inter Medi-A E-Commerce telah berhasil diimplementasikan dan di-deploy di platform Vercel dengan URL: **https://intermedia-store.vercel.app**. Perancangan tampilan antarmuka menggunakan React.js dengan Tailwind CSS untuk responsive design dan user experience yang optimal.

## 1. Halaman Login

Halaman login digunakan oleh pengguna untuk masuk ke dalam sistem sesuai dengan perannya. Pada halaman ini pengguna diminta memasukkan email dan kata sandi yang telah terdaftar.

### Fungsi halaman login:
- **Autentikasi pengguna** dengan validasi email dan password
- **Pembatasan akses berdasarkan peran** (customer, seller, admin)
- **Demo accounts** tersedia untuk testing dengan role berbeda
- **Responsive design** untuk desktop dan mobile
- **Form validation** dengan React Hook Form
- **Loading states** dan error handling

### Fitur Implementasi:
```jsx
// Login Component Features
- Multi-role authentication (Admin, Seller, Customer)
- Form validation dengan react-hook-form
- Password visibility toggle
- Demo account information display
- Gradient branding dengan logo Inter Medi-A
- Responsive layout untuk semua device
```

## 2. Halaman Produk

Halaman produk menampilkan daftar produk yang tersedia dalam sistem e-commerce. Informasi yang ditampilkan meliputi nama produk, harga, stok, dan kategori produk.

### Fungsi halaman produk:
- **Menampilkan katalog produk** dengan grid dan list view
- **Memudahkan customer dalam memilih produk** dengan filter dan search
- **Menyediakan fitur pencarian dan filter produk** berdasarkan kategori, harga, brand
- **Product image handling** dengan fallback dan lazy loading
- **Add to cart functionality** langsung dari product listing
- **Wishlist integration** untuk save produk favorit

### Fitur Implementasi:

// Products Page Features
- Grid/List view toggle

- Sorting options (newest, price, popularity)
- Product image optimization
- Add to cart dengan quantity selector

## 3. Halaman Keranjang dan Checkout

Halaman keranjang digunakan untuk menampilkan produk yang telah dipilih oleh customer sebelum melakukan pemesanan. Selanjutnya, halaman checkout digunakan untuk proses konfirmasi pemesanan dan pembayaran.

### Fungsi halaman keranjang:
- **Menampilkan item yang dipilih** dengan detail produk dan harga
- **Quantity adjustment** dengan increment/decrement buttons
- **Remove items** dari keranjang
- **Calculate total** dengan subtotal dan shipping
- **Empty cart state** dengan call-to-action

### Fungsi halaman checkout:
- **Menampilkan ringkasan pesanan** dengan detail lengkap
- **Menginput alamat pengiriman** dengan form validation
- **Memilih metode pembayaran** (Transfer Bank, E-Wallet, COD)
- **Order confirmation** dengan order tracking number
- **Real-time price calculation** termasuk ongkir

### Fitur Implementasi:
```jsx
// Cart & Checkout Features
- Persistent cart state dengan Zustand
- Quantity management dengan validation
- Price calculation dengan tax dan shipping
- Multiple payment methods
- Address management dengan validation
- Order summary dengan itemized breakdown
- Responsive checkout flow
- Order confirmation dengan tracking
```

### Payment Methods:
- **Bank Transfer**: BCA, Mandiri, BNI, BRI
- **E-Wallet**: GoPay, OVO, DANA, ShopeePay
- **Cash on Delivery**: Untuk area tertentu

**Screenshot**: Cart page dengan item management dan Checkout flow dengan payment selection
