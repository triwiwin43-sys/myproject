## 5.1 Kesimpulan

Berdasarkan hasil penelitian yang telah dilakukan melalui tahapan analisis kebutuhan, perancangan sistem, implementasi menggunakan teknologi React.js dan Node.js, serta pengujian komprehensif sistem e-commerce Inter Medi-A Computer Store, dapat disimpulkan beberapa poin penting sebagai berikut:

### 5.1.1 Kesesuaian Solusi dengan Rumusan Masalah

Penelitian ini berhasil menjawab ketiga rumusan masalah yang telah ditetapkan:

**Rumusan Masalah 1: Kondisi sistem penjualan dan layanan servis yang sedang berjalan**
- Berhasil mengidentifikasi dan mendokumentasikan 7 masalah utama dalam sistem manual melalui analisis menggunakan Rich Picture dan Fishbone Diagram
- Sistem manual yang tidak terintegrasi menyebabkan duplikasi data 15%, kesalahan pencatatan 12 error/hari, dan waktu konfirmasi pembayaran 4-6 jam
- Keterbatasan jangkauan pemasaran hanya melalui WhatsApp dan Instagram membatasi pertumbuhan bisnis

**Rumusan Masalah 2: Perancangan dan implementasi sistem e-commerce berbasis web**
- Berhasil merancang arsitektur sistem 3-tier menggunakan React.js untuk frontend, Node.js untuk backend, dan MongoDB untuk database
- Implementasi 25 komponen React.js yang reusable dengan performance score 94/100 pada Google PageSpeed Insights
- Sistem terintegrasi dengan 20 API endpoints yang mampu menangani 1,200 concurrent users dengan response time rata-rata 320ms

**Rumusan Masalah 3: Pengujian dan evaluasi sistem e-commerce**
- Black Box Testing menunjukkan 96% pass rate (48 dari 50 test cases berhasil)
- Performance Testing membuktikan sistem dapat menangani 1,000 concurrent users dengan throughput 650 requests/second
- Usability Testing menghasilkan SUS score 82/100 (kategori "Excellent") dengan user satisfaction 4.2/5.0

### 5.1.2 Pencapaian Tujuan Penelitian

**Tujuan 1: Analisis sistem penjualan dan layanan servis yang sedang berjalan**
- Berhasil melakukan comprehensive business process analysis menggunakan observasi langsung 40 jam dan wawancara dengan 8 stakeholder
- Identifikasi gap analysis antara kondisi existing dan kebutuhan optimal menggunakan metodologi yang sistematis
- Dokumentasi lengkap proses bisnis manual dengan activity diagram dan rich picture

**Tujuan 2: Perancangan dan implementasi sistem e-commerce terintegrasi**
- Berhasil mengimplementasikan sistem e-commerce full-stack dengan teknologi modern
- Arsitektur cloud-native dengan deployment di Vercel dan MongoDB Atlas mencapai uptime 99.9%
- Fitur lengkap meliputi product management, order processing, payment confirmation, dan real-time tracking

**Tujuan 3: Pengujian komprehensif sistem yang dikembangkan**
- Multi-layered testing strategy dengan Black Box, White Box, Performance, dan Usability Testing
- Validasi fungsionalitas sistem dengan comprehensive test cases dan user acceptance testing
- Evaluasi dampak bisnis dengan before-after comparison menunjukkan improvement signifikan

**Pertumbuhan Bisnis yang Terukur:**

## 5.2 Saran

Berdasarkan hasil penelitian, implementasi, dan evaluasi sistem e-commerce Inter Medi-A Computer Store, serta keterbatasan yang ditemukan selama proses penelitian, berikut adalah saran-saran yang dapat dipertimbangkan untuk pengembangan lebih lanjut:

### 5.2.1 Saran untuk Inter Medi-A Computer Store

**Optimalisasi Sistem Jangka Pendek (1-3 bulan):**

1. **Integrasi Payment Gateway Otomatis**
   - Implementasi Midtrans, Xendit, atau DOKU untuk mengurangi manual verification
   - Estimasi peningkatan conversion rate 25-30% dengan pembayaran yang lebih mudah
   - Pengurangan workload staff untuk verifikasi pembayaran manual
   - **Justifikasi:** Berdasarkan hasil penelitian, 23% customer abandon cart karena proses pembayaran yang kompleks

2. **Pengembangan Mobile Application**
   - Develop aplikasi mobile menggunakan React Native untuk konsistensi teknologi
   - Push notification untuk order updates dan promotional campaigns
   - Offline capability untuk browsing produk tanpa koneksi internet
   - **Target:** Meningkatkan mobile conversion rate dari 2.1% menjadi 4.5%

3. **Advanced Analytics dan Business Intelligence**
   - Implementasi Google Analytics 4 dan Facebook Pixel untuk comprehensive tracking
   - Customer behavior analysis untuk personalized product recommendations
   - Real-time dashboard untuk monitoring KPI bisnis
   - **Manfaat:** Data-driven decision making untuk optimasi marketing dan inventory

**Pengembangan Jangka Menengah (3-6 bulan):**

1. **Marketplace Integration**
   - Integrasi API dengan Tokopedia, Shopee, dan Lazada untuk multi-channel selling
   - Centralized inventory management untuk sinkronisasi stok across platforms
   - Automated product listing dan price synchronization
   - **Potensi:** Peningkatan reach hingga 500% dengan akses ke customer base marketplace

2. **Customer Relationship Management (CRM) System**
   - Implementasi CRM untuk better customer segmentation dan retention
   - Email marketing automation dengan personalized campaigns
   - Loyalty program dengan point system dan tier-based benefits
   - **ROI Expected:** Peningkatan customer lifetime value 40-60%

3. **Advanced Search dan Recommendation Engine**
   - Elasticsearch implementation untuk faster dan more accurate search
   - AI-powered product recommendation berdasarkan browsing history
   - Voice search capability untuk mobile users
   - **Impact:** Peningkatan average order value 15-20% melalui cross-selling

**Strategi Jangka Panjang (6-12 bulan):**

1. **Artificial Intelligence Integration**
   - Chatbot implementation untuk 24/7 customer support
   - Predictive analytics untuk demand forecasting dan inventory optimization
   - Computer vision untuk automated product categorization dan quality control
   - **Benefit:** Pengurangan operational cost 30-40% dengan automation

2. **Omnichannel Commerce Strategy**
   - Integrasi seamless antara online dan offline experience
   - Click-and-collect service untuk customer convenience
   - Augmented Reality untuk virtual product try-on
   - **Goal:** Unified customer experience across all touchpoints

### 5.2.4 Keterbatasan Penelitian dan Mitigasi

**Keterbatasan yang Teridentifikasi:**

1. **Scope Penelitian Terbatas**
   - **Keterbatasan:** Penelitian hanya fokus pada satu studi kasus (Inter Medi-A Computer Store)
   - **Dampak:** Generalisasi hasil mungkin terbatas untuk jenis bisnis atau skala yang berbeda
   - **Mitigasi:** Lakukan comparative study dengan multiple case studies dari berbagai sektor UMKM
   - **Rekomendasi:** Penelitian lanjutan dengan minimum 5 UMKM berbeda untuk validasi framework

2. **Periode Evaluasi Relatif Singkat**
   - **Keterbatasan:** Production monitoring hanya 4 minggu, belum mencakup seasonal variations
   - **Dampak:** Long-term sustainability dan scalability belum dapat divalidasi sepenuhnya
   - **Mitigasi:** Longitudinal study selama 12-18 bulan untuk comprehensive impact assessment
   - **Rekomendasi:** Establish monitoring framework untuk continuous evaluation

3. **Keterbatasan Integrasi dengan Legacy Systems**
   - **Keterbatasan:** Belum ada integrasi dengan existing accounting dan ERP systems
   - **Dampak:** Manual data reconciliation masih diperlukan untuk beberapa proses
   - **Mitigasi:** Develop API integration layer untuk legacy system connectivity
   - **Rekomendasi:** Research tentang legacy system modernization strategies

**Asumsi yang Dibuat dan Validasi:**

1. **Technology Stack Stability**
   - **Asumsi:** React.js, Node.js, dan MongoDB akan remain relevant dalam 3-5 tahun
   - **Risk:** Rapid technology evolution dapat membuat stack obsolete
   - **Mitigasi:** Regular technology assessment dan migration planning
   - **Monitoring:** Quarterly technology trend analysis dan community support evaluation

2. **User Adoption dan Change Management**
   - **Asumsi:** Users akan mudah beradaptasi dengan digital transformation
   - **Reality:** Change resistance dari beberapa stakeholder internal
   - **Mitigasi:** Comprehensive training program dan gradual transition strategy
   - **Learning:** Change management sama pentingnya dengan technical implementation

3. **Scalability Requirements**
   - **Asumsi:** Linear business growth berdasarkan historical trend
   - **Risk:** Exponential growth dapat overwhelm current architecture
   - **Mitigasi:** Implement auto-scaling dan load balancing dari awal
   - **Preparation:** Architecture review setiap 6 bulan untuk capacity planning
